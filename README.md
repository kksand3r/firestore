# 🔥 Firestore Notes App (Flutter + Firebase)

Це проект **Firestore Notes App**, розроблений на Flutter з використанням Firebase для автентифікації користувачів та збереження нотаток у Cloud Firestore. Додаток демонструє базові принципи роботи з Firebase Auth та Cloud Firestore, включаючи реєстрацію, авторизацію, відновлення доступу, а також створення, редагування, читання та видалення нотаток (CRUD).

🌐 **Жива демо-версія:** [https://flutter-auth-app-2f23c.web.app](https://flutter-auth-app-2f23c.web.app)

## 🚀 Основний функціонал

- **Автентифікація (Email/Password):** Реєстрація, вхід в систему та відновлення пароля реалізовано за допомогою пакету `firebase_auth`.
- **База даних нотаток:** Збереження та управління нотатками в реальному часі за допомогою `cloud_firestore`.
- **Захищений доступ:** Доступ до головного екрану з нотатками (`NotesScreen`) можливий лише після успішної авторизації.
- **Маршрутизація стану:** Використовується `AuthWrapper` для автоматичного перенаправлення користувача на екран входу або на головний екран залежно від стану авторизації.
- **Екрани та функції:**
  - 🔐 Екран входу (Login)
  - 📝 Екран реєстрації (Sign Up)
  - 🔄 Відновлення пароля (Forgot Password)
  - 🏠 Головний екран нотаток (NotesScreen з можливістю створення, редагування, видалення нотаток та виходу з акаунту)

## 🛠 Технологічний стек

- **Flutter / Dart** (Фронтенд, UI та логіка)
- **Firebase Authentication** (Логін, реєстрація, управління сесіями)
- **Cloud Firestore** (NoSQL база даних для збереження нотаток)

## 📂 Структура проекту

Основна логіка знаходиться в папці `lib/`:
```text
lib/
  ├─ main.dart                 # Точка входу, ініціалізація Firebase та головний екран нотаток
  ├─ firebase_options.dart     # Конфігурація Firebase
  ├─ models/                   # Моделі даних
  │   └─ note.dart             # Модель нотатки
  ├─ screens/                  # Екрани додатку
  │   ├─ forgot_password_screen.dart
  │   ├─ login_screen.dart
  │   └─ sign_up_screen.dart
  ├─ services/                 # Сервіси
  │   └─ firestore_service.dart # Логіка роботи з Firestore
  ├─ utils/                    # Допоміжні утиліти
  │   └─ auth_errors.dart      # Обробка помилок авторизації
  └─ widgets/                  # Віджети
      └─ auth_wrapper.dart     # Віджет для перевірки стану користувача
```

## 💻 Як запустити локально

1. Отримайте вихідний код проекту.
2. Перейдіть у папку проекту:
   ```bash
   cd firestore
   ```
3. Встановіть залежності:
   ```bash
   flutter pub get
   ```
4. Запустіть додаток (на емуляторі, реальному пристрої або в браузері):
   ```bash
   flutter run
   ```

*Примітка: Проект вже містить базове підключення до Firebase (через `firebase_options.dart`), але для повноцінної роботи вам може знадобитися налаштувати власний проект Firebase за допомогою FlutterFire CLI.*
